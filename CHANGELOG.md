- `2026-07-10` — **Real-time narration: streaming, a synth/play pipeline, filler masking, and fp32.** Made Calliope's narration keep up with playback on CPU — from ~15 s to first word and multi-second gaps down to **~0.01 s to first sound and ~0 s of gaps** on a 30 s+ answer, still `af_nicole` on CPU (no GPU). Layered, each measured: **(1) Streaming** — `pythia.ask()` now streams tokens (`stream: True`, plus `keep_alive: -1` to pin gemma4:e2b resident and a `prewarm()` that preloads it at launch); the home panel renders the answer live and feeds *completed sentences* to Calliope as they form. **(2) Two-stage pipeline** — `speak()` became **non-blocking**: it enqueues sentences, a *synth* worker coalesces the backlog + takes a clause-split chunk + synthesizes it ahead of playback, and a *play* worker drains a bounded audio buffer back-to-back; an epoch counter gives clean barge-in (`stop()`), and stale in-flight synth is dropped. **(3) fp32, not int8 — the key find:** on this CPU the **fp32** model runs at RTF **0.44** (2.3× *faster* than real time) vs int8 at **1.54** — int8's quantize/dequantize overhead was the entire bottleneck, so with fast synth the pipeline structurally pulls ahead of playback and gaps vanish. (DirectML GPU was tried and rejected — kokoro's `ConvTranspose` op is unsupported there on both int8 and fp16; CUDA left as a non-needed option.) **(4) Small chunks** (`chunk_max_chars` 70) so each chunk's synth finishes inside the previous chunk's playback — the actual gap-killer, verified by a prebuffer×chunk-size sweep (cap 70 = 0.0 s gap, cap 140 = 2.6 s). **(5) Prebuffer / lookahead** (`prebuffer_chunks`, default 2) — the first answer chunk waits for a small lead before playback starts (a safety net for bursty LLM streaming); the filler masks the wait, `end_turn()` releases it for short answers. **(6) Filler masking** — filler phrases ("Initiating Cogitator. Please wait.") are synthesized **once at startup** (`prewarm()` → `generate_fillers()`), cached as WAVs in gitignored `calliope_fillers/` (cache key hashes phrase+voice+speed+model, self-heals on change), and played **instantly** (~0.01 s, after a 1 s `filler_delay_ms` beat) while the real answer generates; a guard skips it if real audio already started. Voice **speed 1.0** (was 1.3). All latency knobs live in `calliope_config.json` (`filler_delay_ms`, `prebuffer_chunks`, `first_chunk_max_chars`, `chunk_max_chars`, `fillers`). `tests/test_calliope.py` grew to cover chunking/coalescing, the filler cache round-trip + gate, and prebuffer release (**307 tests OK**, `mypy --strict tools/` green); `tests/test_pythia.py` updated for the streaming response shape + `prewarm`. Docs: `README_PANTHEON/Calliope.md` + `specs/calliope.md` updated for the pipeline/fillers/fp32. *(Builds on the same-day voice-retirement below.)*
- `2026-07-10` — **Voice input retired; Calliope refactored to output-only TTS.** Removed the entire voice-**input** layer and repurposed **Calliope** into a single job: read Pythia's answers aloud, on demand. *Deleted:* `Metis.py` (the mic → Silero VAD → Whisper → Apollo → dispatch → narrate loop), `apollo.py` + `apollo_intents.json` (the keyword router), `metis_config.py`, `tools/metis.py` (the subprocess supervisor), `panels/metis_panel.py` (`MetisLamp`), `tests/test_apollo.py`, `tests/test_metis.py`, and the old `requirements-metis.txt` (the torch/faster-whisper voice stack). The voice-side tool registry in `metis_toolbox/__init__.py` (`TOOLS` + `dispatch()`) is gone — that file is now an empty package marker — so the "three ways in, three registries" design collapsed to **two** (Kairos `WORKERS` + Pythia's reflection-built registry). *Why output-only:* the real dependency weight was the pip **`kokoro`** package (it pulls torch + transformers, gigabytes), not the loop — swapping to **kokoro-onnx** (onnxruntime, no torch) is the actual diet. And voice *input* was the only untrusted command surface — the sole reason Apollo had a frozen 9-tool allowlist and a second registry existed; remove it and there is nothing to route or guard. *New Calliope* (`metis_toolbox/calliope.py`, moved into the toolbox beside `pythia.py`): `narrate(tool_name, result)` became **`speak(text)`** — synthesize an arbitrary string with kokoro-onnx (voice `af_nicole` @ `speed 1.3`, int8 model) and play it through the default output device, **in-process**, never raising. A missing model file or busy audio device degrades to a **logged no-op**, never a crash. Its pure `synthesize(text) → PCM|None` half is the hermetic-test seam. *GUI triggers* (both, the GUI decides *when*, Calliope only makes sound): a per-answer `▶ speak aloud` button in the home chat (`panels/home_panel.py`) **and** a header **narration lamp** (`panels/narrator_panel.py` → `NarratorLamp`) that repurposes the retired voice lamp's slot — a speaker glyph, lit = auto-speak every answer on arrival, dim = silent, off by default. Calliope owns the single source of truth for that auto-speak flag. *Config over code:* voice/speed/lang/model-paths/auto-speak default live in `calliope_config.json`; the kokoro-onnx model binaries (`kokoro-v1.0.int8.onnx` ~88 MB + `voices-v1.0.bin` ~27 MB) sit in `kokoro_models/` (gitignored, downloaded once — see `README_PANTHEON/Calliope.md`). New hermetic `tests/test_calliope.py` (no audio, no model, no network — the kokoro-onnx session is faked and playback is either mocked or driven against a raising stub `sounddevice`, proving graceful degradation). **Verified live** on this box: `af_nicole` int8 synthesizes 2.8 s of 24 kHz PCM for a test line and plays through the default output device without raising; full Felhaven app builds/tears down with the new lamp and no `metis` Kairos worker; the home panel renders the speak button and both manual + auto-speak reach `calliope.speak()`. Docs updated to match: `README_PANTHEON/README.md` (three→two registries), new `README_PANTHEON/Calliope.md` (replaces `Metis.md`), `specs/calliope.md` rewritten (`specs/apollo.md` deleted), `readme.txt`, `CONVENTIONS.md` §2/§8 + decisions log, plus the Pythia/Callimachus/Herodotus/Horai/Scribe module docs. Full suite **283 tests OK**.
- `2026-07-07` — **Callimachus — web search brain tool** (`tools/callimachus.py`), the toolbox's first **multi-tool module**: it exports the plural `TOOL_DEFINITIONS` (a list) with two request-driven brain tools — `search_web(query)` (Brave Search API, one JSON GET over stdlib `urllib`, returns ≤ 3 `{title, snippet, url}` results) and `fetch_page(url)` (https-only, redirects capped at 5 and refused off https, 1 MB download cap, stdlib `html.parser` stripper, text truncated to ~4000 chars with a `truncated` flag). The split is the point: a small quantized model (gemma4:e2b) searches, reads snippets, **chooses one** url, then fetches it — the agentic loop inside one Pythia turn — instead of firehosing whole pages at a 4-bit model. Named for Callimachus of Cyrene, author of the *Pinakes* (the first library catalog, i.e. the search index) — the Zeno/Hypatia/Kepler historical-figure precedent (§12). **Zero new pip deps** (stdlib `urllib`/`json`/`html.parser`/`socket`); neither handler ever raises — every failure is a stable `{"error": ...}` identifier (`no_results` / `rate_limited` / `timeout` / `search_failed:…` / `non_https_url` / `too_large`). The **Brave API key lives in the Cerberus Vault only** (`vault_get('brave_api_key')`, read at call time, never cached to disk, never in the repo/config/env) — so web search works only in a session where Cerberus was unlocked; a locked vault degrades to `search_failed`, not a crash. **Two deliberate deviations from the design handoff**, both because its file list predated the `pythia.py`/`__init__.py` registry split: (1) registered in **`pythia.py`** — the live dashboard-brain registry, whose module-derived `TOOLS`/`_DISPATCH` gained a small `_module_tools()` helper to accept a plural export — **not** `metis_toolbox/__init__.py`, which is the *voice* registry (Apollo → `Metis.py` `dispatch`); wiring web search there would put it on the voice path, which the handoff's own L9 forbids. (2) the five tunables (`_RESULT_COUNT` etc.) are module constants in `callimachus.py` (the aether/zeno convention), **not** `metis_config.py`, which is the voice router's config (Whisper/VAD/Apollo) that no toolbox tool reads. Deliberately no `fetch()`/Kairos worker (request-driven), no panel, no Calliope formatter, no Apollo intent, no query history — each documented in the header so nobody "fixes" the omission. Wiring a `tools/` module to `cerberus.py` for the first time pulled the guardian into the `mypy --strict tools/` graph and surfaced **9 latent annotation errors** there (bare `dict`/`list[dict]` generics + one `str | None` narrowing in `manifest_configs`); fixed in lockstep (annotations only, zero behavior change) so "mypy at zero" stays true and is now actually enforced for `cerberus.py`. New `tests/test_callimachus.py` (24 tests — result shaping/cap, every search + fetch error path, the HTML stripper, the 1 MB cap and Content-Length pre-check, https-only + redirect-to-http rejection, truncation flag, and registry integrity; the single network seam `_OPENER.open` and the key seam `_brave_key` are mocked, so no test touches the network or the Vault); `tests/test_tool_handles.py` bumped 14 → **16** wired Pythia tools.
