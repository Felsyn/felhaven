# Hypatia — The Celestarium

*Anti-Legion: ONE JOB*

Hypatia knows **where every catalog star sits in your sky right now** — and, via
[Kepler](Kepler.md), where the five classical planets are too. It's the amber
**Hypatia — Celestarium** card: a live star map computed from your latitude and
the current sidereal time.

## Catalogs loaded once, math run every tick

Three JSON files load at import (no network, ever):

| File | Holds |
|---|---|
| `hypatia_stars.json` | ~1000 stars: `hip_id → {name, mag, ra, dec}` |
| `hypatia_constellations.json` | constellation line-figures |
| `hypatia_lore.json` | hand-curated blurbs, merged into each constellation once |

The catalogs are generated by `tools/dev_build_hypatia_catalog.py`. Every tick,
Hypatia converts each star's fixed **RA/Dec → Alt/Az** for the observer's
location using sidereal time (JD → GMST → LST → hour angle → alt/az).

## Deliberately low-precision — don't "improve" it

No refraction, no precession, no proper motion. Accuracy is a small fraction of a
degree — invisible at panel scale. The docstring is firm that this is a choice,
not a shortcut. I'm echoing it here because the star math *looks* like something
you'd want to make "more correct," and that would be wasted effort at chart
resolution. The `_altaz(ra, dec, lat, lon, unix)` function is intentionally
generic so **Kepler reuses it verbatim** for planet coordinates — one piece of
spherical trig, two callers.

## Three surfaces (hybrid contract)

Hypatia is the "hybrid" shape (CONVENTIONS §2 — the Morpheus/Metis flavor):

| Surface | Caller | Payload |
|---|---|---|
| **`fetch()`** | Kairos | full snapshot: ~1000 stars + constellations + planets |
| **`set_preset()`** | panel click | mutate latitude, return a fresh snapshot immediately |
| **`handle()`** | LLM (`get_sky_tonight`) | distilled digest — visible planets + brightest 6 stars |

`fetch()` **raises** if the star catalog is empty (total failure) so Kairos
delivers `None` and the panel shows stale. `handle()` never raises — it distills
the ~1000-star snapshot down to "what's notable overhead" (with compass
directions + altitude), because the full catalog would swamp a prompt.

*(The module docstring used to say "NO handle(), NO TOOL_DEFINITION — out of LLM
scope." Stale — `get_sky_tonight` now exists; the docstring has been corrected.)*

## Teaching mode — the latitude presets

`set_preset()` jumps the observer's **latitude** to the North Pole, equator, or
South Pole — but keeps their real **longitude**, so the sky's *timing* stays
honest and only the tilt changes. It's a "what would the sky look like from
there?" teaching toy. The preset is never persisted — every launch wakes on
`current`. It's written from the Tk main thread and read by the Kairos worker
thread; a single string assignment is atomic under the GIL (Morpheus's
concurrency contract), and the ~1000-star recompute is single-digit milliseconds,
so no worker thread is needed.

## Files

| File | Purpose |
|---|---|
| `tools/hypatia.py` | Star positions, the three surfaces, the shared `_altaz` math. |
| `tools/dev_build_hypatia_catalog.py` | Dev-time catalog generator (not runtime). |
| `hypatia_stars/constellations/lore.json` | The catalogs, at the app root. |
| `panels/hypatia_panel.py` → `HypatiaPanel` | The Celestarium card (fixed-width right rail, rescales with the window). |

## Using it

**In the dashboard** — the **Hypatia — Celestarium** card; click a latitude
preset to teleport the sky.

**Ask Pythia** — *"what planets are up?"* / *"what's the brightest star out
right now?"* routes through `get_sky_tonight`.

**Standalone**:

```
python tools/hypatia.py
```
